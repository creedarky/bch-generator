¿Que viene incluido en la carpeta base?

Twitter Bootstrap

Todos los controles bootsrap asi como los estilos en formato LESS se encuentran disponibles para poder usar como principal hoja de estilos. Existe un archivo particular para poder sobreescribir variables y estilos a la necesidad(/src/less/main.less). Para poder actualizar a la ultima version de bootsrap basta con ejecutar el siguiente comando: 

$ cd vendor/twitter-bootstrap
$ git pull origin master

Y estaría listo.

UI Bootstrap

Bootsrap a través de directivas Angular, usando la libreria UI Bootstrap, contiene un set de directivas nativas AngularJS que pueden ser extendidas. Demo:

  • Control Dropdown
    • {{choice}}

No incluye jQuery.

Font Awesome

Es un set de iconos libres que son distribuidos como fuentes. Faciles de usar y livianos. Funciona perfectamente con Bootstrap.

Todos los iconos son creados a través de Font Awesome usando sus clases, un demo: 

<i class="fa fa-flag"></i>

Y obtienes: .

Stylus

Es un framework css que extiende las funcionalidades de css. Esto permite una mejor organización del codigo css. Para más información visita el sitio web de Stylus

Todo el codigo stylus dentro de la carpeta se coloca en el directorio /src/stylus/Mi_archivo.styl. Este codigo es transformado a css y minificado en un archivo css que se adjunta al index.

Filtros

Los filtros pueden ser agregados en el directorio "src/common/filters" y pueden ser utilizadas de la siguiente manera, usaremos como ejemplo el filtro rut localizado en filter.js: 

          { { '1111111-1' | rut } }

genera lo siguiente:

{{'1111111-1' | rut}}

Servicios

Los servicios pueden ser agregados en el directorio "src/common/services" alli puedes encontrar un servicio de muestra. Al inyectar un servicio en tu modulo tu obtienes una instancia de ese servicio.

Ejemplo llamado getRutService: {{fromService}}

Factories

Las factories pueden ser agregadas en el directorio "src/common/factories" alli puedes encontrar una factory de muestra. Al inyectar una factory en tu modulo, tu obtienes el valor que es retornado al invocar la funcion de referencia de la factory.

Ejemplo llamado getRutFactory: {{fromFactory}}

Constantes

La definición de constantes se encuentra en "src/common/constants" alli puedes encontrar constantes que están siendo utilizadas en la base.

Se necesita inyectar el paquete "WebBancoChile.constants" y luego pasarlas como argumento a la función, factoria o servicio deseado. Como ejemplo, las rutas a los datos que poblan el header son leidos desde archivos .json que son referiados a través de constantes. Ver las factorias

  • getDatosUsuarioFactory
  • getProductosUsuarioFActory
  • getUltimoAccesoFactory

Servidor Http

Debes ejecutar el comando "npm start" para poder ejecutar el servidor http.

Pruebas E2E con Protractor

Debes ejecutar el comando "npm run protractor" para poder ejecutar las pruebas escritas en protractor/e2e/scenarios.js
